# 인수 조건: 매거진 상세 페이지

## 시나리오 1: 상세 페이지 표시

**Given** 사용자가 /magazine/reinforcement-guide에 접속했을 때
**When** 페이지가 로드되면
**Then** 해당 매거진의 제목, 카테고리 배지, 날짜가 표시되어야 한다
**And** 전체 콘텐츠 본문이 읽기 편한 레이아웃으로 표시되어야 한다
**And** Breadcrumb 네비게이션이 표시되어야 한다

## 시나리오 2: 관련 콘텐츠 추천

**Given** 상세 페이지가 표시된 상태에서
**When** 하단으로 스크롤하면
**Then** 동일 카테고리의 관련 콘텐츠가 최대 3개 표시되어야 한다
**And** ContentCard 컴포넌트를 통해 일관된 UI로 표시되어야 한다

## 시나리오 3: 관련 콘텐츠 없음

**Given** 동일 카테고리에 다른 콘텐츠가 없는 경우
**When** 상세 페이지를 로드하면
**Then** 관련 콘텐츠 추천 섹션이 표시되지 않아야 한다

## 시나리오 4: 404 처리

**Given** 사용자가 존재하지 않는 slug로 접근했을 때
**When** 페이지가 로드되면
**Then** 404 페이지가 표시되어야 한다
**And** 매거진 목록 페이지로 이동할 수 있는 링크가 제공되어야 한다

## 시나리오 5: Breadcrumb 네비게이션

**Given** 상세 페이지에서
**When** Breadcrumb의 "매거진" 링크를 클릭하면
**Then** /magazine 목록 페이지로 이동해야 한다

## 시나리오 6: SEO 메타데이터

**Given** 매거진 상세 페이지가 로드되면
**When** 페이지 소스를 확인하면
**Then** 고유한 title, description, og:image 태그가 존재해야 한다

## 시나리오 7: 관련 콘텐츠 클릭

**Given** 관련 콘텐츠가 표시된 상태에서
**When** 관련 콘텐츠 카드를 클릭하면
**Then** 해당 콘텐츠의 상세 페이지로 이동해야 한다

## 품질 게이트

- TypeScript 타입 에러 없음
- ESLint 경고 없음
- Dynamic Route 정상 동작
- SEO 메타데이터 각 페이지마다 고유하게 생성
- 이미지 최적화 (Next.js Image 사용)
- 모바일 읽기 편의성 확인
- 접근성: Breadcrumb에 aria-label 적용
- 404 페이지 정상 동작
