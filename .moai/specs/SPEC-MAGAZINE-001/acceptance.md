# 인수 조건: 매거진 목록 페이지

## 시나리오 1: 전체 목록 표시

**Given** 사용자가 /magazine 페이지에 접속했을 때
**When** 페이지가 로드되면
**Then** 전체 매거진 콘텐츠가 그리드 레이아웃으로 표시되어야 한다
**And** 각 콘텐츠는 ContentCard 형태로 표시되어야 한다
**And** "전체" 필터가 기본으로 선택되어 있어야 한다

## 시나리오 2: 카테고리 필터링

**Given** 전체 콘텐츠가 표시된 상태에서
**When** "카드뉴스" 필터를 클릭하면
**Then** 카드뉴스 카테고리의 콘텐츠만 표시되어야 한다
**And** "카드뉴스" 필터 칩이 활성화 상태로 표시되어야 한다
**And** 페이지 새로고침 없이 즉시 필터링되어야 한다

## 시나리오 3: 필터 초기화

**Given** 특정 카테고리가 필터링된 상태에서
**When** "전체" 필터를 클릭하면
**Then** 모든 카테고리의 콘텐츠가 다시 표시되어야 한다

## 시나리오 4: 빈 상태 처리

**Given** 사용자가 목록 페이지에서
**When** 콘텐츠가 없는 카테고리를 선택하면
**Then** 적절한 빈 상태 메시지가 표시되어야 한다

## 시나리오 5: 콘텐츠 카드 클릭

**Given** 매거진 목록이 표시된 상태에서
**When** 사용자가 콘텐츠 카드를 클릭하면
**Then** 해당 매거진의 상세 페이지로 이동해야 한다

## 시나리오 6: 반응형 디자인

**Given** 사용자가 다양한 디바이스에서 접속했을 때
**When** 화면 크기가 모바일(<640px)이면
**Then** 콘텐츠가 1열로 표시되어야 한다
**When** 화면 크기가 태블릿(640-1024px)이면
**Then** 콘텐츠가 2열로 표시되어야 한다
**When** 화면 크기가 데스크톱(>1024px)이면
**Then** 콘텐츠가 3-4열로 표시되어야 한다

## 품질 게이트

- TypeScript 타입 에러 없음
- ESLint 경고 없음
- 모바일/태블릿/데스크톱 반응형 동작 확인
- 접근성: 키보드 네비게이션 가능
- 접근성: ARIA labels 적용
- 기존 ContentCard 컴포넌트 재사용
